# 🏠 전세사기 예방 AI 플랫폼 (JeonseGuard)
> **졸업작품 겸 실서비스 프로토타입 프로젝트 (2025)**  
> 전세계약서의 위험 요소를 AI로 자동 분석하고, 실거래가·등기·보증·신탁 등의 공공 데이터를 연동하여  
> **전세사기 위험을 사전에 경고하는 웹 기반 플랫폼**입니다.

---

## 🌟 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 전세사기 예방 AI 플랫폼 (JeonseGuard) |
| **개발 목적** | 전세계약 체결 전, 사용자가 계약서를 업로드하면 자동으로 위험 조항과 실거래 리스크를 분석하여 사기 피해를 예방 |
| **프로젝트 성격** | 졸업작품 + 실서비스 수준 시제품 (정식 론칭 가능성 있음) |
| **핵심 가치** | 법률Tech + 부동산Tech 융합을 통한 사회적 가치 창출 |

---

## 🧩 주요 기능

| 구분 | 기능명 | 설명 |
|------|---------|------|
| 📤 **전자문서 업로드** | 사용자가 전세계약서(PDF 등)를 업로드 |
| 🔍 **Lawform AI 분석** | 계약서 전체를 Lawform AI에 전송 → 위험 조항, 불공정 문구 자동 감지 |
| 🤖 **GPT 리포트 생성** | Lawform 결과를 GPT 프롬프트를 통해 일정한 형식의 요약·리포트로 정리 |
| ⚠️ **하이라이트 표시** | 계약서 내 위험 문구를 색상(빨강/주황/노랑)으로 표시 및 툴팁 제공 |
| 🗂️ **리포트 다운로드** | 리스크 요약, 근거 문구, 권장 행동이 포함된 리포트(PDF/HTML) 생성 |
| 🧾 **공공기관 링크 제공** | 등기정보·전입신고·보증 등 공식 페이지로 바로 이동 가능한 버튼 제공 |
| 🧑‍⚖️ **법률 상담 연계** | 위험건 발생 시 변호사 상담 연결 API (로펌 제휴 연동 가능) |
| 🔒 **보안 처리** | 문서 업로드 후 즉시 Lawform으로 전송, 전송 완료 시 서버 내 파일 삭제 |
| 🧹 **데이터 최소 보관** | 분석 종료 후 데이터 즉시 삭제 (사용자 페이지 이탈 시 복구 불가) |

---

## 🔁 웹 진행 방식 (Flow)

사용자 → PDF 업로드
↓
[문자 추출 / 보안검사]
↓
Lawform AI 분석 요청
↓
Lawform 결과(JSON)
↓
GPT 프롬프트로 리포트 생성
↓
위험 문구 색상 표시 + 리포트 제공
↓
사용자가 결과 확인 후 페이지 종료 → 모든 데이터 삭제

yaml
코드 복사

**핵심 포인트**
- 계약서 자체는 서버에 영구 저장되지 않음  
- 분석 과정은 Lawform AI가 법률 판단 담당  
- GPT는 리포트 형태로 요약 및 시각화만 담당  
- 보안·면책·법률고지 포함된 UX 설계

## ⚙️ 기술 스택

| 영역 | 기술 |
|------|------|
| **Frontend** | React / Next.js / Tailwind / PDF.js |
| **Backend** | Node.js (Express) or FastAPI |
| **AI Layer** | Lawform AI + OpenAI GPT-5 API |
| **Data Storage** | PostgreSQL / S3 (문서 임시 저장 후 삭제) |
| **OCR / 텍스트 처리** | PaddleOCR / PDFMiner / LayoutLMv3 |
| **API 연동** | 국토부 실거래가, HUG, CODEF(등기정보), 정부24 링크 |
| **Deployment** | AWS / Cloudflare / Docker |
| **Security** | TLS + AES256 암호화 / 자동삭제 정책 / 면책고지 표시 |
